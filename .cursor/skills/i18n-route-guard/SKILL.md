# Охорона маршрутів i18n

## До появи реалізації

- MVP може бути однією локаллю; цей skill актуальний після появи `[locale]` або подібного.

## Принципи

- **Єдиний вхід**: middleware задає локаль; узгоджена стратегія префікса або домену.
- **Fallback**: невідома локаль → дефолтна або 404 за рішенням продукту.
- **Посилання**: навігація з урахуванням локалі; уникай захардкоджених шляхів, що ламають не дефолтні локалі.
- **SEO**: `hreflang` і canonical для перекладених публічних сторінок.

## Чеклист

- [ ] Дашборд і публічне бронювання коректні для всіх підтримуваних локалей
- [ ] Немає змішаної локалі в одній сесії без явної дії користувача
- [ ] Дата/час форматуються з узгодженим часовим поясом (користувач чи бізнес)
