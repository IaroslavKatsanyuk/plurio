# Supabase RLS

## Принципи

- **За замовчуванням заборона**: увімкни RLS на таблицях з даними користувачів; додавай явні політики.
- **Скоуп власника**: політики через `auth.uid()` і/або колонки `user_id`, що відповідають авторизованому користувачу.
- **Публічне бронювання**: окремі політики для анонімних або обмежених операцій — вузькі колонки й рядки.
- **Service role**: лише на сервері; ніколи в браузер; обходить RLS — використовуй рідко в довіреному серверному коді.

## Чеклист

- [ ] Кожну політику перевірено для дозволених і заборонених випадків
- [ ] Немає політики, що витікає всі рядки, коли `user_id` null
- [ ] Міграції зворотні або задокументовані, якщо ні
