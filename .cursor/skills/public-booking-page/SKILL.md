# Публічна сторінка бронювання

## Принципи

- **Скоуп**: відвідувачі створюють/читають лише те, що власник бізнесу відкрив (доступність, послуги); без доступу до даних інших користувачів.
- **Резолюція**: `[username]` (або slug) мапиться на один профіль бізнесу; невідомий slug — 404.
- **Створення**: записи через `appointment.service` (або еквівалент); RLS дозволяє insert лише для задуманих рядків (наприклад прив’язка до id власника).
- **Зловживання**: rate limits і захист від ботів — коли з’являться в продукті; усі вхідні дані валідуй на сервері.

## Чеклист

- [ ] Внутрішні id чи email в URL лише якщо це навмисно й безпечно
- [ ] Повідомлення успіху/помилки не розкривають, чи існує email у системі
- [ ] Часовий пояс для слотів явний (TZ бізнесу проти TZ користувача) за правилами продукту
