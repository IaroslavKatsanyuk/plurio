# Аудит кодової бази

## Порядок рев’ю

1. **Архітектура**: дотримано service layer? Supabase лише за сервісами/lib?
2. **Дані**: RLS на всіх таблицях з даними користувачів; міграції версійні; типи згенеровані або узгоджені зі схемою
3. **Auth**: middleware і сесії узгоджені; без витоку секретів
4. **Публічні поверхні**: `/u/[username]` і API віддають лише задумані поля
5. **DX**: скрипти lint/typecheck; приклад env задокументовано

## Формат виходу

- **Критично**: треба виправити перед масштабом / чутливо до безпеки
- **Важливо**: варто виправити найближчим часом
- **Бажано**: опційне прибирання

Заверши **готовністю до міграції** (1–2 речення): чи service layer готовий замінити виклики Supabase на HTTP?
