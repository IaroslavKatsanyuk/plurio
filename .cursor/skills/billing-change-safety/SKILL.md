# Безпека змін біллінгу

## Принципи

- **Ідемпотентність**: обробники webhook за ключем події провайдера; безпечні повтори.
- **Джерело істини**: статус підписки з провайдера або синхронізованого сховища; UI відображає стан сервера після синхронізації.
- **Entitlements**: обмеження функцій читаються з сервера (сервіс або API), не лише з клієнтських прапорців.
- **Секрети**: секрет підпису webhook лише на сервері; ніколи в клієнтських бандлах.

## Чеклист

- [ ] Webhook перевіряє підпис перед зміною стану
- [ ] Зміни плану оновлюють БД у транзакції, якщо зачіпається кілька таблиць
- [ ] Невдалі платежі й скасування мають визначені стани для користувача
- [ ] Test і live ключі чітко розділені в env
