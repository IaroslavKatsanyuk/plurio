# Патерн API / Route Handler

## Принципи

- Надавай перевагу функціям **service layer** для доменної логіки; файли маршрутів лишай тонкими (розбір запиту → сервіс → відповідь).
- **Відповіді**: стабільна форма JSON для успіху (`{ data }` або доменна) і помилок (`{ error: { code, message } }`) — узгоджуй з наявними handlers у репо.
- **Коди статусів**: 2xx успіх, 400 валідація, 401/403 auth, 404 не знайдено, 409 конфлікт, 429 rate limit, 5xx неочікуване.
- **Секрети**: перевіряй підписи для webhooks; не логуй сирі токени чи PII.

## Чеклист

- [ ] Вхід валідовано (схема або ручні перевірки) перед викликом сервісу
- [ ] Контекст auth визначено (сесія / API key) там, де потрібно
- [ ] Помилки не витікають stack trace або внутрішніми id клієнтам у production
- [ ] `export const dynamic` / кешування враховано за потреби
