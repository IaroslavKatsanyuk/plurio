# Security Hotspots Hook

## Мета

При роботі з чутливими зонами — не пропустити типові дірки для SaaS на Next.js + Supabase.

## Коли застосовувати

Торкаєшся: автентифікації, сесій, cookies, `middleware`, env, Supabase ключів, RLS, публічного бронювання (`/u/[username]`), webhooks, Telegram / нотифікацій.

## Кроки

1. **Ізоляція даних**: доступ до рядків лише в межах `user_id` (або майбутнього tenant); перевірка на рівні RLS, не лише в UI.
2. **Ключі**: `service_role` / секрети — лише server-only; ніколи в `NEXT_PUBLIC_*` і не в клієнтських бандлах.
3. **Публічні форми**: валідація вводу, обмеження спаму (rate limit / captcha — коли з’явиться в продукті), мінімум витоку PII у відповідях.
4. **Редіректи**: open redirect після логіну — не довіряти довільним query-параметрам без allowlist.
5. **Інтеграції** (Telegram тощо): токени в env; підпис або секрет для webhook; ідемпотентність повторних викликів де потрібно.
